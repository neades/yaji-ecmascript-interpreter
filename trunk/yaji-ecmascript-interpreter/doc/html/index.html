FESI (pronounced like fuzzy)  is a full implementation of the first version of the EcmaScript language (defined in the standard ECMA 262 available at http://www.ecma-international.org (edition of june 97).  EcmaScript is largely equivalent to the JavaScript language version 1.1 or to the core part of JScript, but without the navigator specific extensions.

Capabilities:

  • FESI implements most of the specified standard (there are small discrepancies dues to the grammar and naturally some bugs, but it is not a toy implementation).
  • FESI currently implements only the first version of the standard (roughly Javascript 1.1). It does not include the significant extensions of version 2.
  • FESI is easy to extend in a compatible way (without changing the grammar). A set of extensions are provided they are:
    • basic input/output ,
    • file input/output ,
    • access to Java objects (including dynamic loading of classes and beans).
    • database access and
    • regular expression based search from an EcmaScript program.
  • FESI can be used from the Bean Scripting Framework.
  • FESI can run on any version of Java since JDK 1.1.8, with the exception of some extensions requiring external libraries.


The documentation of FESI is included here. It is part of the kit too.

FESI consists of a set of java packages , allowing to use EcmaScript as a macro language for Java applications, and of an interactive interpreter (usable from the command line to test EcmaScript programs). The integration with Java is very strong, making FESI a useful tool to test Java libraries.

There is a simple editor included with the Swing version of the interpreter. You can execute scripts interactively or from the editor windows.

A few examples of code:

  • Simple code to list system properties , an example to access Java from EcmaScript.
  • The juggler code creates the juggler bean with start and stop button. An example how a bean can be tested from an EcmaScript program.
  • An example of using Swing from EcmaScript, including event handling.
  • A simple XML parser written entirely in EcmaScript.
